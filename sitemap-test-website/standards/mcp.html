Model Context Protocol (MCP)

Model Context Protocol（MCP）是由 Anthropic 於 2024 年底發起的開放通訊協定，旨在建立 AI 模型與外部資料來源、工具之間的標準化連接方式。MCP 經常被比喻為 AI 領域的「USB-C」— 一個通用的介面標準。

設計動機

在 MCP 出現之前，每個 AI 應用都需要針對每個外部服務建立專屬的整合方案，這導致了大量重複工作與碎片化的生態系。MCP 透過標準化的協定解決了以下問題：

  • N×M 問題：N 個 AI 應用 × M 個工具 = N×M 個自訂整合 → MCP 將其簡化為 N+M。
  • 重複建設：每次整合都要重新設計認證、錯誤處理、資料格式等基礎架構。
  • 互通性不足：不同 AI 平台的工具定義格式互不相容。

核心架構

MCP 採用客戶端 - 伺服器（Client-Server）架構：

元件 角色 說明
MCP Host 宿主應用 AI 應用程式（如 Claude Desktop、IDE 外掛）
MCP Client 客戶端 在 Host 內部，負責維護與 Server 的連接
MCP Server 伺服器 對外暴露工具、資源與提示詞的輕量服務

三大核心功能

  • Tools（工具）：讓 AI 模型呼叫外部功能，例如查詢資料庫、操作檔案系統。與 Claude 的 Tool Use 功能緊密整合。
  • Resources（資源）：讓 AI 模型讀取外部資料，例如文件內容、API 回應。
  • Prompts（提示詞範本）：預定義的提示詞模板，可跨應用共享與重用。

傳輸方式

MCP 支援多種傳輸協定：

  • stdio：透過標準輸入/輸出進行本地通訊，適合本機工具。
  • Streamable HTTP：透過 HTTP 進行遠端通訊，支援串流回應。

生態系現況

MCP 已獲得廣泛的產業支持，以下是部分已整合 MCP 的平台與工具：

  • Claude Desktop — Anthropic 官方桌面應用
  • VS Code / GitHub Copilot — 微軟的開發環境
  • Cursor — AI 原生程式碼編輯器
  • Zed — 高效能程式碼編輯器
  • Sourcegraph — 程式碼搜尋與智慧分析平台
