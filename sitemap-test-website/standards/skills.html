Skills（技能模組）

Skills 是 Copilot / Claude 編輯器整合中的一項可擴充機制，允許使用者為 AI 助手定義特定領域的專業知識模組。每個 Skill 封裝了一組指令與上下文，讓模型在觸發時能夠以領域專家的方式執行任務。

Skill 是什麼

Skill 本質上是一份結構化的指令文件（通常命名為 SKILL.md），包含：

  • 名稱與描述：定義 Skill 的用途與觸發條件，讓 AI 知道何時該使用此技能。
  • 詳細指令：具體的執行步驟、規則、輸出格式等，引導模型完成特定類型的任務。
  • 工具限制：可指定 Skill 執行時允許或禁止使用的工具（如 Terminal、檔案編輯等）。
  • 適用範圍：透過 applyTo 模式匹配特定檔案類型或專案結構。

Skill 的結構範例

# SKILL.md

---
name: code-review
description: |
  Review code changes for quality, security, and performance.
  Trigger when user asks to "review", "audit", or "check" code.
tools:
  - read_file
  - grep_search
  - semantic_search
---

## 指令

1. 先閱讀目標檔案的完整內容
2. 檢查安全性問題（OWASP Top 10）
3. 檢查效能瓶頸
4. 檢查程式碼風格一致性
5. 產出結構化的審查報告

Skill 與其他機制的比較

機制 範圍 觸發方式 用途
Skills 特定任務 AI 自動匹配或使用者指定 封裝領域專業知識
System Prompts 全域 每次對話自動載入 定義模型基礎行為
Agents 特定角色 使用者選擇或子代理呼叫 定義完整的 AI 角色模式
MCP Tools 外部服務 AI 判斷後呼叫 連接外部資料與功能

常見的 Skill 類型

程式碼審查

針對 Pull Request 或特定檔案進行安全性、效能與風格的自動化審查，產出結構化的審查報告。

測試生成

根據既有程式碼自動生成對應的單元測試，遵循專案的測試框架與命名慣例。

文件撰寫

以特定格式（如 JSDoc、Sphinx）為函式、模組或 API 端點產出文件。

工作流自動化

定義多步驟的開發流程，如 Spec-Driven Development、月報生成、Issue 分析等。

Skill 的放置位置

Skills 可以放在不同層級，影響其作用範圍：

  • 使用者層級（~/.copilot/skills/）：跨所有專案生效，適合個人通用的技能。
  • 專案層級（.copilot/skills/ 或 .github/copilot/skills/）：僅在該專案中生效，適合團隊共享的專案特定技能。

與 MCP 的協作

Skills 可以搭配 MCP 工具使用。例如，一個「Jira 月報」Skill 可以呼叫 Jira MCP Server 取得資料，再根據 Skill 中定義的格式規範產出報告。這種組合讓 AI 同時具備「知道怎麼做」（Skill）和「能夠執行」（MCP Tool）的能力。
