System Prompts（系統提示詞公開）

Anthropic 是業界首家完整公開其 AI 產品系統提示詞（System Prompt）的公司。這項舉措旨在提高 AI 系統的透明度，讓使用者可以了解模型在回答時所遵循的指導方針。

什麼是 System Prompt

System Prompt 是在每次對話開始時注入給 AI 模型的一段隱藏指令，用於設定模型的行為框架。它通常包含：

  • 模型的身份定義與角色描述
  • 行為準則與限制條件
  • 輸出格式要求
  • 能力邊界的說明
  • 安全相關的護欄設定

為何公開 System Prompt

傳統上，AI 產品的系統提示詞被視為商業機密，不對外公開。Anthropic 選擇公開的原因包括：

  • 透明度：使用者有權了解影響 AI 行為的核心指令，這有助於建立信任。
  • 可審計性：外部研究者和公眾可以檢視和評估 AI 系統設計中可能存在的偏見或問題。
  • 推動產業標準：希望引領業界朝向更透明的方向發展。
  • 教育價值：公開的提示詞可作為開發者設計自己 AI 系統的參考範本。

System Prompt 的結構

一個典型的 Claude System Prompt 包含以下區塊：

  1. 身份宣告：聲明模型名稱、開發者與版本資訊。
  2. 核心行為準則：定義安全、誠實與實用性的優先順序。
  3. 能力描述：列出模型可以做什麼、不能做什麼。
  4. 格式指引：規範回應的結構、長度與風格。
  5. 工具使用規則：若啟用 Tool Use，定義工具呼叫的時機與方式。
  6. 安全護欄：列出明確禁止的行為類型。

對開發者的啟示

如果你正在使用 Claude API 建構自己的應用，設計良好的 System Prompt 至關重要。以下是幾點建議：

  • 明確定義 AI 的角色與預期行為
  • 設定清楚的安全邊界與拒絕條件
  • 指定輸出格式以確保一致性
  • 參考 Anthropic 公開的 System Prompt 作為範本
  • 根據 Model Spec 理解模型的預設行為
